# Git Rules

## Never put AI attribution in commits

Commits represent user decisions. No "Co-Authored-By: Claude", no "Generated by", no AI signatures. Ever. AI contributions are logged only in `contrib.txt`.

## Workflow

```bash
# From /opt/csc (always commit from repo root)
git pull origin main
refresh-maps              # ALWAYS before commit — stale maps send agents to dead ends
git add -A
git commit -m "brief description of what changed and why"
git push origin main
```

See `.csc_root` for exact push/pull/commit commands as hints.

## refresh-maps is mandatory

Run `refresh-maps` before every commit. It regenerates:
- `tools/INDEX.txt` + per-package `.txt` files — code API maps
- `docs/tree.txt` — directory tree
- `docs/p-files.list` — flat file listing

Without it, next agent gets stale maps and wastes time on dead code paths.

## Branch workflow for risky changes

```bash
git checkout -b feature/description
# ... make changes ...
git add -A && git commit -m "description"
git push -u origin feature/description
gh pr create --title "..." --body "$(cat <<'EOF'
## Summary
<what this PR does>

## Workorder
<paste the full workorder content>

## Work Log
<paste the WIP journal content>
EOF
)"
# PR requires review by the pr-reviewer agent before merge
```

Direct commits to main are fine for small, safe changes. Feature branches + PR for anything touching server core, queue_worker, pm, or shared library.

## PR body is mandatory — include workorder + work log

**Every PR must include in its body:**

1. **Summary** — what the PR does in 1-3 bullets
2. **Workorder** — the full text of the workorder that assigned this task
3. **Work log** — the full WIP journal from `wo/wip/` (your step-by-step log)

This is not optional. The PR reviewer agent uses these to verify that the code
matches what was requested and that the work was done as described. A PR without
a workorder and work log will be rejected automatically.

```bash
# Convenience: include files in PR body
gh pr create \
  --title "brief description" \
  --body "$(printf '## Summary\n<bullets>\n\n## Workorder\n```\n%s\n```\n\n## Work Log\n```\n%s\n```' \
    "$(cat wo/wip/YOURWORKORDER.md)" \
    "$(cat wo/wip/YOURWORKORDER.md)")"
```

## No force push to main

Never `git push --force` to main. If you need to fix a commit, add a new one.
